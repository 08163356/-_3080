1.前端如何避免布局偏移

```
在前端开发中，布局偏移通常是由于页面中的元素尺寸或位置发生变化导致的。为了避免布局偏移，可以采用以下一些技巧：

避免使用动态尺寸：尽量避免使用动态尺寸的元素，例如使用百分比、em、rem 等单位来设置元素的宽度、高度、边距等属性。这些单位会根据父元素或根元素的尺寸变化而变化，从而导致元素的尺寸和位置发生变化。

使用 CSS 盒模型：在设置元素的尺寸和边距时，使用 CSS 盒模型，即将元素的边框、内边距和内容尺寸分开计算。这样可以避免元素尺寸的计算错误，从而导致布局偏移。

避免使用绝对定位：尽量避免使用绝对定位来布局元素，因为它会使元素从文档流中脱离，从而影响其他元素的位置和尺寸。如果必须使用绝对定位，可以使用相对定位的父元素来控制元素的位置和尺寸。
<!-- more -->

避免使用 JavaScript 修改元素尺寸和位置：尽量避免使用 JavaScript 直接修改元素的尺寸和位置，因为这可能会导致页面的重排和重绘，从而影响页面性能。如果必须修改元素的尺寸和位置，可以使用 CSS3 的 transform 属性，它可以在不影响布局的情况下修改元素的位置和尺寸。

预留空间：在设计页面布局时，可以预留一些空间，以便在内容发生变化时能够容纳新的内容，从而避免布局偏移。

总的来说，避免布局偏移需要注意元素的尺寸和位置，并尽量避免直接修改元素的尺寸和位置。
```
